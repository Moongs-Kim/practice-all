[git 권장 사항과 커밋 메세지 컨벤션]

<커밋 권장 사항>
- 하나의 커밋에는 한 단위의 작업을 넣는다.
  문제가 발생했을 경우 코드 변경 사항을 추적해서 이전 상태로 쉽게 돌아갈 수 있다.
- 여러 개의 변경 사항도 서로 다른 커밋으로 나누느 것이 좋다.
  그러면 해당 변경 사항만 쉽게 찾아 되돌릴 수 있다.

<컨벤션>
커밋 메세지는 변경 사항의 내용과 의도를 명확하게 표현해야 한다.

컨벤션이란 : 개발 팀원들끼리 일관된 방식으로 커밋 메세지를 작성하여 프로젝트의 품질과
           유지 보수성을 높이기 위한 규칙을 말한다.
           컨벤션은 보편적으로 정해진 규칙이 아니라 팀의 업무에 맞춰 팀원들이 합의한 것이다.
