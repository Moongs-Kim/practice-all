[git 헝크 단위로 스테이징]

헝크(hunk)란 덩어리란 뜻이다.
깃에서 하나의 파일에서 수정된 코드 블록의 일부를 말한다.

git add -p : 헝크별로 변경 사항이 나타나며 각 헝크의 변경 여부를 받아들일지 물어본다.
             하나의 파일에서 부분적 add 가능

헝크 변경 여부 옵션(자주 사용되는 명령어)
y : 헝크를 스테이지 영역으로 add 한다.
n : 헝크를 스테이지 영역으로 add 하지 않는다.
q : add를 종료한다.
s : 현재의 형크를 더 작은 단위의 헝크로 나눈다.
e : 현재의 형크를 수동으로 편집한다.

git commit -v : 커밋 메세지를 입력하라는 메세지와 어떤 변경 사항이 반영되었는지 볼 수 있다.
