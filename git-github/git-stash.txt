[git 스태시]

스태시란 : 작업하던 내용을 깃에서 다른 공간에 잠시 치워 두는 기능

* 스태시를 하려면 먼저 깃의 관리 대상(tracked)이 되어야 한다.

git stash : 현 작업들 치워 두기
    git stash save 이지만 git stash만 입력해도 가능

git stash apply : 치워 둔 마지막 항목(번호 없을 시) 적용
                  끝에 번호로 항목 지정 가능 ex) git stash apply stash@{1}
git stash drop : 치워 둔 마지막 항목(번호 없을 시) 삭제
                 끝에 번호로 항목 지정 가능 ex) git stash drop stash@{1}
git stash pop : 치워 둔 마지막 항목(번호 없을 시) 적용 및 삭제
                apply + drop
git stash branch 브랜치 명 : 새 브랜치를 생성하여 pop
                            충돌 사항이 있는 상황에 유용
git stash clear : 치워 둔 모든 항목들 비우기

git stash -p : 변경된 여러 파일 중 원하는 부분만 골라 스태시 가능
               맨 마지막에 수정한 내역부터 반영 여부 물어본다. (y/n 누르면됨)
git stash -m '메세지': 메세지와 함께 스태시

git stash list : 스태시 목록을 보여준다. (번호 + 메세지)
                 번호를 가지고 원하는 시점에서 해당 스태시를 프로젝트에 다시 적용할 수 있다.
* 스태시를 적용한다는 것은 커밋을 한다는 뜻이 아니다.
* 파일을 그 상태로(스태시한 상태)로 저장하고(add안됨) 그 후 add, commit 등 원하는 작업 진행
* 즉 파일을 다시 수정된 상태로 돌려 놓는다는 말
